#### I. Acteurs
1. Utilisateur (Généralisation)
   - Locataire : Réserve des résidences, consulte les disponibilités, laisse des avis.
   - Gestionnaire : Gère les résidences, les disponibilités, les tarifs, les prestations, et le matériel.
2. Système de Paiement : Gère les paiements sécurisés.
3. Système d'envoi de mail : Gère les envois de mail en fonction de l'état du système

--

#### II. Cas d'Utilisation

##### Module d'Authentification
- Se connecter : Tous les utilisateurs doivent se connecter pour accéder au système.
- Se déconnecter : Les utilisateurs peuvent se déconnecter.

##### Module de Réservation
- Réserver une résidence :
  - Inclut : "Vérifier les disponibilités", "Payer une réservation".
  - Étend : "Laisser un avis" (après la fin de la location).
- Annuler une réservation : Le locataire peut annuler une réservation sous certaines conditions.
- Modifier une réservation : Le locataire peut modifier les détails d'une réservation.
- Consulter les disponibilités : Le locataire consulte les disponibilités des résidences.

##### Module de Gestion
- Gérer les résidences : Le gestionnaire ajoute, modifie ou supprime des résidences.
- Gérer les disponibilités : Le gestionnaire définit les périodes de disponibilité et les contraintes.
- Gérer les tarifs : Le gestionnaire définit les tarifs pour chaque résidence.
- Gérer les prestations : Le gestionnaire ajoute, modifie ou supprime des prestations.
- Gérer le matériel : Le gestionnaire ajoute, modifie ou supprime du matériel.
- Gérer les utilisateurs : Le gestionnaire gère les comptes des locataires.

##### Module de Paiement
- Payer une réservation : Le système de paiement gère le paiement sécurisé.

##### Module de Communication
- Envoyer des e-mails : Le système envoie des e-mails automatiques (confirmations, rappels, etc.).

##### Module d'Avis
- Laisser un avis : Le locataire laisse un avis après la fin de sa location.
- Consulter les avis : Les locataires peuvent consulter les avis laissés par d'autres utilisateurs.

---

### III. Relations Détaillées

#### a. Inclusion (Include)
- Réserver une résidence → Vérifier les disponibilités : Avant de réserver, le système vérifie les disponibilités.
- Réserver une résidence → Payer une réservation : Le paiement est obligatoire pour confirmer une réservation.

#### b. Extension (Extend)
- Réserver une résidence → Laisser un avis : Le locataire peut laisser un avis seulement après la fin de la location.

#### c. Héritage (Généralisation)
- Utilisateur → Locataire : Un locataire est un type d'utilisateur.
- Utilisateur → Gestionnaire : Un gestionnaire est un type d'utilisateur.
